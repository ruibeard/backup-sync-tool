Feature inventory

Every label, button, indicator, and state the main window must support (paired + unpaired). Pairing popup is a separate modal.

Always visible (paired main window)

AreaElementNotes
Status strip required Left accent bar + dot + text Connection only — not sync progress. Text: Connected, Offline, Not paired, Reconnect required, Pairing · waiting for approval. Hover shows server URL + destination tooltip.
Server row required SERVER heading · hostname · grey button Button: Connect (unpaired) / Reconnect (paired or auth failure). Grey 76×26 px, same style as Open/Browse.
Local folder required Label Backup folder on this PC Editable path field + Open + Browse. Auto-save on browse.
Destination required Label + read-only panel Paired: Server destination + flat grey panel (#fafafa). Unpaired: Destination folder (hint from XD detection).
Recent activity required RECENT ACTIVITY owner-draw list Per-file rows with mini progress bars. Failed uploads pinned top (red + error detail). Empty: No recent activity.
Sync footer required Batch progress + status text All synced · Checking… · Syncing · done/total · pct% · ETA · N upload(s) failed · Retry failed button (when paths known).
Settings required Two checkboxes Start with Windows (default on) · Sync from server (maps to download-from-server). Auto-save on toggle.
Meta footer required Version · GitHub · author Version link (e.g. v2026.0.3) · GitHub icon button · conditional blue Update when newer release exists · Rui Almeida link right.
Explicitly absent — No Save button. No blocking MessageBox for routine status (ribbon + activity only).
